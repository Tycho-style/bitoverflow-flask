Event Guidelines

Duration: Two Weeks

BitOverflow​ is an online event encouraging contributions to ​Open Source Software​. The contributions in the form of code will be accepted at the official repository of ​bitoverflow​ hosted on the code platform ​GitHub​.

It is to be noted that:

  • Participants can contribute to the repositories of their own choice
  • Participants can contribute to multiple repositories

Some repositories will be specially targeted at beginners, e.g. ​submit code for printing ‘hello world’ in different languages​, while others will be accepting meaningful code contributions to existing projects e.g. ​preparing tkinter gui for an existing console app​. ​Mentors​ will be assigned to each project and they will be in-charge examining and accepting contributions. ​Chat channels shall be arranged for interaction and co-ordinations between participants and mentors of each project.

At the end of the event, digital certificates shall be awarded to the participants based on their contribution value and count. ​No bitoverflow certificates shall be awarded to participants with no contributions till the end of the event. ​Participants can receive the following achievements on the certificates based on their contributions.

  • Top Code​ - If participants is among Top 5 contributors
  • Bit Knight​ - awarded for valuable and meaningful code contributions
  • Markdown Master​ - awarded for writing clear and comprehensible documentation

Criteria for meaningful contributions:

The below given criteria shall be relaxed to a degree for beginner repositories

  • Commits containing only ​code re-writes, comments, new-lines​ and ​spaces​ will ​not​ be accepted
  • Commits containing documentation should be written in ​markdown​ and should be comprehensible
  • Commits containing code should be properly ​indented​ and free from syntax errors
  • Pull Requests should describe properly the ​changes​ made in the commits
